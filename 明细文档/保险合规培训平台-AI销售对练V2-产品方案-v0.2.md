## 功能单元 1：V2入口与开练配置

### 功能概述
降低开练门槛，支持用户在首页、课程完成页或训练记录页快速进入对练；不强制要求先完成课程。

### 业务流程

```text
用户进入首页/课程页/训练记录页
      ↓
点击「立即开始对练」
      ↓
选择主题（默认推荐）
      ↓
选择对练方式（文本 / 语音）
      ↓
选择难度（初级/标准/挑战）
      ↓
开始对练
```

### 业务规则

**触发条件**
- 用户已登录且具备对练访问权限。
- 不校验“是否已完成课程”，直接可开练。

**处理逻辑**
1. 默认推荐 1 个训练主题。
2. 默认对练方式为“文本”，用户可切换“语音”。
3. 点击开始后创建会话ID并进入会话页。

**边界情况**
- 语音权限被拒绝：自动回退文本模式并提示。
- 麦克风初始化失败：保留会话，允许手动切文本继续。
- 推荐主题不可用：降级到通用训练主题。

### 原型设计（已选定）

**方案 A：一键开练（已选）**

```text
┌────────────────────────────┐
│ AI销售对练 V2              │
├────────────────────────────┤
│ 推荐主题：重疾险需求沟通    │
│ 对练方式：文本 / 语音       │
│ 难度：标准  [修改]          │
│                            │
│ [   立即开始对练   ]        │
│ [ 查看其他主题 ]            │
└────────────────────────────┘
```

### 功能点描述

#### V2入口与开练

**入口路径**:
- 首页 → AI对练
- 课程完成页 → 去对练
- 我的 → 训练记录 → 再练一次

**页面名称**: "AI销售对练V2-开练页"

**页面功能**:
1. **推荐主题区**:
   - 展示推荐主题与简介
   - 支持切换主题
2. **对练模式区**:
   - 文本/语音二选一
   - 语音模式需调用麦克风权限
3. **操作按钮**:
   - 立即开始对练：创建会话并跳转
   - 查看其他主题：进入主题列表

**业务规则**:
- 不做“先学后练”强制校验。
- 语音模式失败时不阻断开练，自动回退文本。
- 同一用户最多同时存在 1 个进行中会话。

**待补充内容**:
1. 主题推荐算法版本规则
2. 语音时长上限与超时策略

### 验收标准

- [ ] 用户可不经过课程页面直接开练。
- [ ] 开练页可展示文本/语音模式切换。
- [ ] 语音权限拒绝时可自动回退文本并继续。
- [ ] 点击开始后成功创建会话并进入会话页。
- [ ] 主题不可用时系统可降级到通用主题。

## 功能单元 2：自由式无脚本会话引擎（文本+语音）

### 功能概述
在无固定脚本下，由语义识别 Agent 和进程推进 Agent 协同完成动态对练；同时支持文本输入和语音输入。

### 业务流程

```text
AI客户开场
   ↓
用户文本输入 / 语音输入
   ↓
(语音模式) ASR转文本
   ↓
语义识别Agent：意图/风险/有效性
   ↓
进程推进Agent：追问/转场/收束
   ↓
高风险命中？
   ↓是
本回合结束后提示纠偏（不打断）
   ↓
下一回合
```

### 业务规则

**触发条件**
- 每次用户提交内容（文本或语音转写文本）触发一次识别与推进。

**处理逻辑**
1. 语音输入先转写，再参与识别与推进。
2. 风险命中采用“回合后提示”策略：
   - 本回合先完成AI正常回复。
   - 回复后追加风险提示卡与建议表述。
3. 会话在以下任一条件结束：目标达成 / 达到最大轮次 / 用户主动结束。

**边界情况**
- ASR失败：提示“未听清”，用户可重录或改文字输入。
- 连续2轮低质量回答：触发引导提示。
- 高频高风险命中：提示频率限流（每2轮最多1次）。

### 原型设计（已选定）

**方案 A：单栏沉浸式（已选）**

```text
┌────────────────────────────┐
│ 客户画像：35岁二孩家庭      │
│ 目标：保障+预算平衡         │
├────────────────────────────┤
│ AI客户：我担心买了用不上... │
│ 你：...                    │
│ AI客户：...                │
│ [回合后提示]               │
│ 风险等级：中               │
│ 证据句："..."             │
│ 建议表达："..."           │
├────────────────────────────┤
│ [按住说话] [文本输入框]     │
│ [发送] [结束本次训练]       │
└────────────────────────────┘
```

### 功能点描述

#### 对练会话（双模输入）

**触发操作**:
- 用户发送文本
- 用户完成语音录入

**页面名称**: "AI销售对练V2-会话页"

**页面功能**:
1. **对话流区域**:
   - 展示AI与用户多轮对话
   - 展示当前客户画像与训练目标
2. **输入区域**:
   - 文本输入与语音输入并存
   - 语音输入支持重录
3. **回合后风险提示卡**:
   - 显示风险等级（低/中/高）
   - 显示证据句与建议表达

**业务规则**:
- 风险提示不得打断当前回合主对话。
- 风险结论必须绑定证据句。
- 所有识别标签和提示结果写入日志。

**待补充内容**:
1. 语音识别供应商与准确率阈值
2. 风险提示文案标准库

### 验收标准

- [ ] 会话页支持文本和语音输入。
- [ ] 语音可正常转写并参与对话推进。
- [ ] 高风险命中时采用回合后提示，不中断当前回合。
- [ ] 风险提示卡包含等级、证据句、建议表达。
- [ ] ASR失败可重录或切文本继续。
- [ ] 会话日志含输入模式、风险标签、推进决策。

## 功能单元 3：训练总结与复练

### 功能概述
输出结构化复盘结果，报告仅展示“等级 + 证据 + 建议”，避免分数焦虑并提高可执行性。

### 业务流程

```text
训练结束
   ↓
总结Agent生成报告
   ↓
展示：总体等级 + 问题等级 + 证据句 + 建议
   ↓
用户选择：针对问题再练 / 返回首页
```

### 业务规则

**触发条件**
- 会话结束后自动触发总结。

**处理逻辑**
1. 总体结论使用等级制：优秀 / 达标 / 待提升。
2. 问题项使用等级制：高风险 / 中风险 / 低风险。
3. 每个问题项必须包含：证据句 + 建议表达。

**边界情况**
- 证据不足：标记“待复核”，不输出强判断。
- 总结失败：异步重试并通知用户稍后查看。

### 原型设计

```text
┌────────────────────────────┐
│ 本次训练结果：达标          │
├────────────────────────────┤
│ 问题1（中风险）            │
│ 证据句："..."             │
│ 建议表达："..."           │
│ [针对该问题再练一次]       │
│                            │
│ 问题2（低风险）            │
│ 证据句："..."             │
│ 建议表达："..."           │
└────────────────────────────┘
```

### 功能点描述

#### 总结报告页

**入口路径**:
- 会话结束自动跳转
- 我的 → 训练记录 → 查看报告

**页面名称**: "AI销售对练V2-总结报告页"

**页面功能**:
1. **总体结论区**:
   - 展示本次训练等级
2. **问题清单区**:
   - 按风险等级排序展示问题
   - 每条问题展示证据句和建议表达
3. **复练操作区**:
   - 针对问题再练一次
   - 返回首页

**业务规则**:
- 不展示数值分。
- 高风险问题固定置顶。
- 无证据不展示问题结论。

**待补充内容**:
1. 等级定义细则（优秀/达标/待提升）
2. 同主题历史趋势展示策略（是否纳入本期）

### 验收标准

- [ ] 报告仅展示等级，不展示分数。
- [ ] 每个问题都有证据句。
- [ ] 每个问题都有建议表达。
- [ ] 高风险问题优先展示。
- [ ] 支持从报告一键发起复练。

- 依据产品上下文《产品定位》：主链路为年度强制学习，AI对练作为创新探索，不影响主链路。
