## 产品定位

**系统名称**：灾害信息员培训与竞赛SaaS平台

**核心定位**：参考已上线国家级学习与竞赛平台的业务模型与能力清单，重新建设一套完全独立的多租户培训+竞赛SaaS平台；新平台在代码、数据、部署、账号体系上均独立运行，通过“平台标准能力 + 数据配置”模式，低成本支持各省开展本地化灾害信息员培训与竞赛。

**系统关系边界**：
- 已上线国家级学习与竞赛平台：独立历史系统，仅提供可借鉴的业务流程与能力经验。
- 本SaaS平台：独立新建系统，不是国家级平台的升级版、子系统或同库同部署延伸。
- 资产复用范围：产品方案、题库结构、运营方法等可迁移资产；最终以本SaaS重新实现与独立治理为准。
- 操作边界：本系统内仅省级及以下组织进行业务操作；国家级单位不作为本系统直接操作角色。

**治理/运营模式**：
- 参考来源阶段：2024年国家级学习与竞赛平台已上线并交付国家减灾中心使用（经验来源，非本系统）。
- 参考赛事阶段：2025年国家级平台支撑首届全国灾害信息员业务技能大赛（经验来源，非本系统）。
- 新建SaaS阶段（当前目标）：基于可迁移业务资产，重新建设独立SaaS平台，面向各省客户提供租户化服务，我方以配置数据为主完成交付与运营支撑。

**目标用户**：
- **省级应急管理主管单位**：组织培训、发布考试、查看统计结果。
- **地市县管理人员**：落实培训执行、人员管理、考试组织与过程督导。
- **灾害信息员学员**：完成学习、练习、考试、参赛与成长积分。
- **平台运营与实施团队（我方）**：完成租户开通、数据配置、内容运营和技术保障。

**核心场景**：
- **常态化培训场景**：按年度培训要求组织课程学习、题库练习和达标考评。
- **赛事组织场景**：按个人赛/团体赛、多阶段赛制组织线上与线下联动比赛。
- **监管督导场景**：按地区、层级、个人查看学习与考试数据，支撑监督和通报。
- **SaaS交付场景**：新省份客户签约后快速完成租户开通与本地化数据配置并上线运营。

## 商业模式

**商业模式定位**：
- **公司定位**：应急管理数字化场景的解决方案与平台服务商。
- **核心业务**：灾害信息员培训与竞赛平台能力输出（平台+内容+运营）。
- **核心客户**：省级应急管理相关单位及其下属管理体系。
- **业务聚焦**：多租户SaaS能力、标准化培训与赛事编排能力、低成本配置化交付能力。

**收费模式**：
- **省级客户**：以平台服务采购为主（可按年服务费/项目制，具体合同策略待确认）。
- **增值服务**：课程内容共建、专项运营活动、竞赛保障服务（按需增购）。

**核心价值主张**：
- **对省级主管部门**：快速具备“培训+竞赛+监管”一体化能力，形成量化管理抓手。
- **对地市县执行单位**：减少线下组织与人工统计压力，提升执行效率和可追踪性。
- **对灾害信息员**：随到随学、随学随练、考后即反馈，形成持续成长闭环。
- **对我方运营**：通过标准化平台与配置化交付，实现跨省复制与规模化服务。

**战略原则**：
- 平台标准能力统一建设，省级差异通过“数据与规则配置”实现。

**核心竞争力与商业壁垒**：
- **壁垒本质**：国家级项目落地经验 + 真实业务场景验证 + 赛事组织能力沉淀。
- **价值闭环**：
  - 对客户：政策落实可量化 + 培训效果可评估 + 赛事组织可复用。
  - 对平台：学习数据与赛事数据沉淀，持续优化内容和运营策略。
- **竞争力构建路径**：
  - **短期竞争力**：省级快速交付（开通即用、配置即运营）。
  - **长期壁垒**：多省实践数据沉淀形成培训与竞赛运营方法库。

**风险规避策略**：
- 避免重定制开发，优先配置化交付，控制项目复杂度与交付风险。
- 保持核心规则的统一与版本化管理，防止多租户规则分裂。

**对功能设计的影响**：
- 功能优先级需优先支持“快速开通、低运维、可复制”。
- 所有关键流程必须支持按租户独立配置与审计追踪。

## 术语表

| 术语 | 定义 | 易混淆概念 |
|------|------|------------|
| 灾害信息员 | 在国家灾害信息员体系内承担灾情相关统计报送与基层协同职责的人员 | 与普通应急志愿者不同，具备明确业务职责与管理要求 |
| 学分 | 基于课程学习完成情况获得的培训量化指标 | 与“积分”不同，学分偏培训达标管理 |
| 积分 | 基于签到、学习任务等行为获得的激励指标，可用于运营激励 | 与“学分”不同，积分偏活跃度与激励 |
| 线上练习 | 面向日常学习巩固的题库练习过程 | 与正式“线上比赛/考试”不同，通常不作为正式竞赛成绩 |
| 线上比赛 | 按赛制规则开展的正式线上竞赛过程 | 与线下现场比赛共同组成完整赛事体系 |
| 团体赛 | 按县/市/省等层级组队进行的赛事形式 | 与个人赛并行，不可互相替代 |
| 租户 | SaaS模式下的独立客户空间（通常为省级单位） | 与组织层级（省/市/县）不同，租户是平台隔离单位 |
| 数据配置 | 我方以配置方式完成行政区划、组织、人员、课程、题库、赛制等上线准备 | 与定制开发不同，强调标准能力复用 |

### 业务分类枚举（可选）

#### 竞赛阶段（5类）

1. 练习阶段：线上练习与课程学习。
2. 初赛阶段：按规则进行线上初赛。
3. 复赛阶段：按规则进行线上或线下复赛。
4. 决赛阶段：进行同级别决赛。
5. 总决赛阶段：省级队伍线下总决赛。

#### 赛事层级（4类）

1. 基层层级
2. 县级层级
3. 市级层级
4. 省级层级

## 模块清单

### 学习端（首页/学习/考练/我的）（新SaaS建设项，参考能力已验证）

**定位**：面向灾害信息员的日常学习与测评入口。  
**核心场景**：学习任务承接、课程学习、题库练习、个人成长记录。  
**使用角色**：灾害信息员学员。  
**主要流程**：
```text
登录平台 → 查看学习任务 → 学习课程 → 练习/考试 → 获得学分积分 → 查看个人档案
```
**与其他模块的关系**：
- ← 内容与题库管理模块：提供课程与试题内容
- → 培训统计模块：沉淀学习与考核行为数据

### 监管端（培训统计/考评管理/人员管理）（新SaaS建设项，参考能力已验证）

**定位**：面向主管和执行单位的培训监管与考评管理中台。  
**核心场景**：统计总览、地区统计、学员详情、考试发布与统计、人员入离职管理。  
**使用角色**：省级主管管理员、地市县执行管理员。  
**主要流程**：
```text
发布培训/考试任务 → 监控执行进度 → 查看统计与明细 → 调整人员与任务配置
```
**与其他模块的关系**：
- ← 学习端：接收学习与考试数据
- → 竞赛管理模块：复用人员与考评数据

### 竞赛管理模块（新SaaS建设项，参考能力已验证）

**定位**：支撑多阶段、多层级个人赛与团体赛组织执行。  
**核心场景**：赛程编排、参赛资格管理、在线对局/排名、晋级与结果发布。  
**使用角色**：省级主管管理员、地市县执行管理员、学员。  
**主要流程**：
```text
创建赛事 → 配置赛制与时间窗 → 组织比赛 → 计算排名/晋级 → 输出赛事结果
```
**与其他模块的关系**：
- ← 人员管理与组织数据：用于参赛资格与分组
- ← 题库管理：用于竞赛题目供给
- → 统计分析模块：沉淀竞赛过程与结果数据

### 多租户与租户运营模块（❌ 待规划，SaaS化关键）

**定位**：支撑一套平台服务多个省级客户，确保数据与权限隔离。  
**核心场景**：租户开通、租户参数管理、租户生命周期管理。  
**使用角色**：我方运营人员、平台超级管理员。  
**主要流程**：
```text
创建租户 → 配置组织与权限模板 → 绑定内容与赛制模板 → 开启租户运行
```
**与其他模块的关系**：
- → 全部业务模块：提供租户边界和配置上下文

### 数据配置中台（❌ 待规划，低运维交付关键）

**定位**：把各省差异收敛为配置项，减少定制开发。  
**核心场景**：行政区划导入、组织层级配置、人员导入、课程包和题库包装载、赛事规则配置。  
**使用角色**：我方运营人员、省级管理员。  
**主要流程**：
```text
选择租户 → 导入/编辑配置数据 → 校验发布 → 生效验证
```
**与其他模块的关系**：
- → 学习端/监管端/竞赛模块：提供业务运行所需配置数据

### 内容与题库中台（🚧 部分实现）

**定位**：统一管理课程内容、题目资产和版本。  
**核心场景**：课程分层分类、题库组卷策略、省级内容差异配置。  
**使用角色**：内容运营、省级主管管理员（部分权限）。  
**主要流程**：
```text
内容入库 → 分类标签化 → 课程/试卷发布 → 使用反馈回流优化
```
**与其他模块的关系**：
- → 学习端：供课
- → 考评与竞赛：供题

## 数据底座

> 平台数据底座以“人员组织、内容题库、学习考评、竞赛结果、租户配置”五类数据为核心，支撑培训与竞赛全链路。

### 人员与组织信息库

**定位**：作为平台权限与业务分发的基础数据源。  
**核心内容**：
- 租户信息：租户ID、租户状态、开通时间。
- 组织信息：省/市/县/乡镇等层级结构。
- 人员信息：学员身份、角色、在岗状态、审核记录。

### 课程与题库资产库

**定位**：统一承载课程内容、题目资产与版本。  
**核心内容**：
- 课程资源：课程分类、章节、时长、发布版本。
- 题库资源：题型、难度、知识点标签、题目版本。
- 组卷规则：考试模板、竞赛组题策略。

### 学习与考评过程库

**定位**：沉淀学习过程与考评结果，用于监管与复盘。  
**核心内容**：
- 学习行为：学习时长、完成率、签到记录。
- 考试结果：场次成绩、错题记录、通过状态。
- 学分积分：学分累计、积分变更、排名快照。

### 竞赛运营数据域

**定位**：承载赛事组织与结果数据。  
**核心内容**：
- 赛事配置：阶段、赛制、时间窗、晋级规则。
- 赛事过程：对局记录、轮次成绩、积分变化。
- 赛事结果：个人/团体排名、获奖与晋级结果。

### 数据协同价值

- 形成“配置-执行-统计-复盘”闭环，支持持续优化。
- 为省级客户提供可量化管理依据，支持通报和改进。
- 支撑SaaS规模化复制，降低跨省交付成本。

## 设计原则

- **政策约束优先**：培训与竞赛能力必须服务政策落实，不做偏离主责业务的功能扩展。
- **配置优先于定制**：优先通过数据和规则配置满足省级差异，避免重复开发。
- **租户隔离刚性**：跨租户数据、权限、日志严格隔离，审计可追溯。
- **结果可量化**：学习、考试、竞赛、组织执行均需可统计、可对比、可复盘。
- **低门槛可执行**：面向基层与执行人员，流程简洁、反馈及时、移动端优先。

## 功能规划原则

### 优先级判断标准

功能优先级按以下顺序判断：

1. **政策合规与业务闭环** > 数据安全与租户隔离 > 配置化交付效率 > 体验优化
- 先满足监管与赛事刚性要求，再提升运营效率和体验。
- 先解决“能用、可管、可追溯”，再解决“更好用”。
- 先做平台级标准能力，再做省级个性化增强。

2. **必须实现功能（SaaS化底座）**
- 多租户与权限隔离。
- 数据配置中台（组织、人员、课程、题库、赛制）。
- 培训与竞赛核心闭环（发布-执行-统计-复盘）。

3. **优先实现功能（规模化运营）**
- 省级模板化开通（标准初始化包）。
- 租户运营看板（活跃、达标、参赛、风险预警）。
- 运营工单与问题闭环。

4. **重点打磨功能（效果提升）**
- 学员学习路径优化与错题强化。
- 竞赛组织效率工具（自动分组、自动晋级校验）。
- 统计报表可视化与导出能力。

5. **适度投入功能（增值能力）**
- 激励玩法扩展（积分商城深化等）。
- 智能推荐与智能运营辅助（在不影响主链路前提下迭代）。

### 功能投入策略

- **平台底座能力**：持续高投入，保证安全、稳定、可复制。
- **省级配置化能力**：优先投入，直接影响交付效率和毛利。
- **个性化增值能力**：按客户价值与复用性评估，谨慎投入。
