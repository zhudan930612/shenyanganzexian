# 调研对象清单（融合版，按分类，含新技术边界）

## 使用说明
- 本文档已融合：`调研对象清单-保险应急高匹配版` + `调研对象清单-跨行业分维度版`。
- 展示方式：按“特定分类”分组，不再按文档来源拆开。
- 在进入案例调研前，先做“新技术边界初筛”，避免把普通数字化能力混入新技术池。

## 新增字段说明
- 阶段：第一阶段（保险垂直）/ 第二阶段（跨行业发散）
- 优先级：P0（优先）/ P1（次优先）/ P2（观察）
- 边界结论：纳入 / 待补证据 / 不纳入（按“进入主流程+有技术决策/自动化+可指标验证+可复用”判定）
- 成熟度：L1 概念验证 / L2 试点运行 / L3 稳定运行
- 建议先调研模块：用于后续按模块填写《案例详细记录模板》

## 建议执行顺序（含边界门）
1. 第0步：先做边界初筛，优先处理“纳入/纳入（待核）”对象。
2. 第1步：先做第一阶段 P0（1、2、3、6），形成保险垂直结论。
3. 第2步：再补第一阶段 P1（4、5、7、8），完善可迁移性判断。
4. 第3步：第二阶段优先“销售赋能+高合规”（11-15），最后补教育科技参考（18-20）。

## 证据采集最小标准
- 每个对象至少收集 3 条证据。
- 每个对象至少覆盖 2 类来源（A/B/C）。
- 优先采用近12个月信息。
- 每条证据必须尽量绑定模块ID，避免“对象级空泛结论”。
