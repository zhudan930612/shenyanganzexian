# 培训学习领域新技术应用调研框架

## 1. 调研目标
- 支撑全年路线图：识别保险行业培训学习中可落地的新技术方向。
- 输出可执行的学习形态升级方案，并形成试点建议。
- 明确“做类似功能的可行性与代价”，支撑后续立项判断。
- 在学习效果提升之外，识别可缓解单一盈利模式压力的潜在创新方向。

### 1.1 新技术边界定义（必须先判定再调研）

#### A. 纳入边界（满足以下全部条件才算“新技术”）
1. 进入学习主流程：影响“学/练/评/管”至少一个核心环节，而非外围展示功能。
2. 存在技术决策或自动化能力：如模型推理、推荐策略、智能评测、检索增强、行为预测等。
3. 能被指标验证：至少绑定一个学习或业务指标（如完课率、留存、学习频次、转化辅助指标）。
4. 可复用：不是一次性活动脚本，能够在不同批次或人群重复运行。

#### B. 排除边界（出现任一项则不纳入“新技术”）
1. 仅UI改版、页面美化、文案调整。
2. 仅把线下流程搬到线上，但没有新增智能决策或自动化能力。
3. 仅采购通用工具但未接入学习主流程（仅试用、未规模化使用）。
4. 仅停留在宣传口径，缺少可追溯的实际应用证据。

#### C. 成熟度边界（用于优先级）
- L1 概念验证：有原型或小样，未稳定运行。
- L2 试点运行：已在真实人群小范围使用。
- L3 稳定运行：已规模化使用并持续迭代。
- 调研优先顺序：L3 > L2 > L1。

## 2. 本产品背景与评估前提（固定口径）

### 2.1 业务背景
- 产品名称：栗子同学（保险从业人员合规培训平台）。
- 业务定位：保险销售从业人员执业合规培训。
- 用户规模：约 200 万保险从业人员用户。
- 运行周期：已稳定运行 8 年。
- 当前状态：业务可正常运转，但留存率极低，用户主要在政策要求节点集中学习。
- 政策约束：保险从业人员每年必须完成至少一次合规学习（政策强制要求）。
- 当前盈利模式：以课程购买为主要盈利点。
- 行业变化：保险从业人员规模下降，课程购买增长空间承压。
- 本轮机会点：在“年度必学”基础上，探索可持续使用的新培训形态。

### 2.2 本轮双目标
- 目标A（必保）：让用户在低障碍条件下完成合规必学课程，降低年度必学流失。
- 目标B（增量）：通过新学习形态辅助业务员销售转化。
- 目标关系：先保证 A，再放大 B。

### 2.3 决策与执行前提
- 路线图属性：全年滚动规划（当前无硬性截止日期）。
- 决策原则：根据具体场景决策，不做“一刀切”技术路线。
- 实施策略：先复现功能，再通过测试尽可能复现结果。
- 数据基础：已有可用数据资产，可支持行为分析与试点验证。
- 预算前提：暂未设置预算上限，需通过代价分级控制投入节奏。
- 商业导向：优先考虑能提升留存、复用频次或形成新增价值点的方向。

### 2.4 失败标准（当前定义）
- 原则定义：没有用户使用，即视为失败。
- 执行建议（便于落地）：试点期应补充量化口径（如激活率、次月留存、周使用频次）。

## 3. 调研范围与顺序
1. 第一阶段（先收敛）：保险垂直对象（专业培训机构、行业认证机构、保险公司/渠道体系）。
2. 第二阶段（再发散）：跨行业对象（销售赋能、高合规培训、一线运营、教育科技）。

## 4. 迁移与试点决策口径（用于路线图）

### 4.1 可迁移判定四问
1. 该对象的技术能力是否进入学习主流程（不是宣传层）。
2. 在我方场景中是否有可复用的触发-反馈-复盘闭环。
3. 我方现有数据、内容、系统是否能支撑最小可行版本（MVP）。
4. 预估代价是否与全年路线图节奏匹配。
5. 是否对“低留存 + 单一课程购买盈利”问题产生实质改善潜力。

### 4.2 代价估算分级（统一口径）
- 低代价：以采购/配置为主，小规模集成；周期 1-2 个月；跨系统改造少。
- 中代价：采购 + 二次开发或局部自研；周期 2-4 个月；需要多角色协同。
- 高代价：核心能力自研或深度重构；周期 4 个月以上；涉及核心系统改造。

### 4.3 路线图排序规则
- P0：迁移可行 + 代价低/中 + 有明确使用场景，且对留存或商业价值有直接改善潜力。
- P1：价值明确但依赖条件未齐或代价偏高，先小范围验证。
- P2：证据弱、迁移难或代价不匹配，持续观察。

### 4.4 试点评估最小要求
- 每个试点必须定义：目标人群、核心场景、周期、成功阈值、失败阈值。
- 每个试点必须回答：是否有人用、是否持续用、是否带来目标指标变化。

## 5. 单对象统一分析框架

### 5.1 对象画像
- 对象定位、服务人群、核心学习场景。
- 与保险培训的场景映射（新人上岗/销售转化/合规复训/主管带教）。

### 5.2 新技术应用清单
- 主要使用的技术：模型能力、推荐引擎、知识库/RAG、语音识别、视频分析、评测引擎等。
- 技术来源：自研 / 第三方 API / SaaS 平台 / 混合方案。
- 应用位置：学习前 / 学习中 / 学习后。
- 解决问题：完课障碍、掌握度、迁移率、管理效率中的哪一项。

### 5.3 实现逻辑
- 场景触发：什么人、在什么情况下触发该能力。
- 输入：依赖哪些数据、内容、行为信号。
- 处理：核心规则/模型如何工作（推荐、打分、问答、反馈等）。
- 输出：对学员、主管、系统产生什么结果。
- 业务动作：输出结果如何进入训练、评估、复盘闭环。

### 5.4 资源支撑要求
- 数据资源：学员行为数据、题库数据、业务行为数据、标签体系。
- 内容资源：课程内容、话术脚本、案例库、标准答案与评分标准。
- 技术资源：产品研发、算法/提示工程、数据工程、平台运维、系统集成。
- 组织资源：业务专家、培训运营、管理者机制、跨部门协同。
- 合规资源：隐私保护、数据权限、审计留痕、模型输出风控。

### 5.5 代价预估
- 评估路径：自研 / 采购 / 混合（采购 + 二次开发）。
- 一次性投入：人月、外采费用、接入改造成本。
- 持续投入：模型调用、算力与存储、运营维护、内容更新。
- 实施周期：PoC（4-8周）/ 试点（2-4月）/ 全量（4月以上）。
- 输出分级：低代价 / 中代价 / 高代价（用于是否跟进决策）。

### 5.6 机制拆解（重点）
- 触发条件：什么情况下启动该机制。
- 行为变化：学员和管理者的行为如何变化。
- 闭环逻辑：如何从“学”走到“练”“评”“改”。
- 结果信号：可观察到的外部结果。

### 5.7 结果证据（粗证据口径）
- 证据分级：A（量化公开）/ B（多案例一致）/ C（市场信号）。
- 最低标准：每个对象至少 3 条证据、至少 2 类来源。
- 证据要素：来源、时间、摘要、可信度、可追溯链接。

### 5.8 保险可迁移评估
- 可直接迁移点。
- 需改造迁移点（流程、系统、组织、数据、合规）。
- 主要风险与前置依赖。

### 5.9 优先级结论
- P0：可立即试点（证据较强 + 改造成本可控）。
- P1：可观察并小范围验证（证据中等或依赖待补齐）。
- P2：先观察（证据弱或不具备当前落地条件）。

## 6. 输出物清单
1. 调研对象清单（已完成）。
2. 单对象案例卡（使用《案例详细记录模板》逐个填写）。
3. 汇总结论页（P0/P1/P2 + 试点建议）。
4. 年度路线图建议（按低/中/高代价分层）。

## 7. 执行建议
- 先完成第一阶段 P0 对象案例卡，再进入 P1/P2。
- 每完成 3 个对象做一次横向复盘，及时修订评分口径。
- 每个对象必须输出“是否可做 + 代价等级 + 推荐路径（自研/采购/混合）”。
- 每个候选能力必须给出“首个试点场景”和“失败退出条件”。
