# 产品原型 HTML 生成规则

> 本文档定义了从功能设计文档（.md）生成 HTML 原型的完整规则。
> **用途**：批量生成可交互的 HTML 原型，供 PM 演示、评审、与开发沟通使用。
> **交付标准**：原型需达到"开发直接可用"水平——开发人员无需额外询问 PM，单凭 HTML 文件即可理解功能意图、页面结构、交互逻辑、字段规则和边界状态。

---

## 一、输入文档说明

### 必读文档（按优先级）

1. **功能设计文档**（主输入）
   - 路径：`明细文档/功能设计文档/[模块名]功能设计.md`
   - 提取内容：功能单元列表、原型设计章节、交互说明、页面状态说明

2. **产品上下文**（补充背景）
   - 路径：`明细文档/[产品名]产品上下文.md`
   - 提取内容：产品名称、用户角色定义、整体导航结构

3. **产品规划**（可选）
   - 路径：`明细文档/[产品名]产品规划.md`
   - 提取内容：模块列表（用于生成侧边导航）

### 提取规则

从功能设计文档中，**按顺序**提取以下内容生成原型：

| 设计文档章节 | 提取用途 |
|---|---|
| `## 功能单元 N：[名称]` | 生成一个独立页面（Tab 或路由） |
| `#### AI 初稿方案` / `#### PM 迭代原型` | **优先取 PM 迭代原型**；PM 未填写则用 AI 推荐方案（AI 推荐字段标注的那个） |
| `**交互说明**` 表格 | 映射为 JS 交互行为 |
| `**页面状态说明**` | 生成状态切换控件 |
| `### 功能点描述` > 字段清单 | 映射为表单字段和数据列 |
| `### 验收标准` | 在页面底部生成"验收要点"折叠面板（灰色，不影响主视图） |

## 二、HTML 输出规范

### 2.1 文件结构

每个功能设计文档生成 **1 个 HTML 文件**（单文件，无外部依赖）：

```
[模块名]原型.html
```

文件内嵌所有 CSS 和 JS，可直接用浏览器打开，无需启动服务。唯一外部依赖：Mermaid CDN（流程图渲染，需联网）。

### 2.2 响应式断点

| 断点名 | 屏幕宽度 | 对应设备 |
|---|---|---|
| Desktop | ≥ 1280px | PC 浏览器 |
| Tablet | 768px – 1279px | 平板、小屏 PC |
| Mobile | < 768px | 手机浏览器 / 微信内嵌页 |

HTML 中通过 `@media` 实现响应式，**不引入外部 CSS 框架**，全部内联样式。

---

### 2.3 PC 端页面整体布局

采用**三栏布局**：左侧导航 + 主内容区 + 右侧标注区（可折叠）。

```
┌──────────────────────────────────────────────────────────┐
│  [产品名] · [模块名] 原型              v草稿 🤖AI稿  [查看标注] │
├──────────┬───────────────────────────┬────────────────────┤
│          │                           │                    │
│  左侧     │  主内容区                 │  标注面板（可折叠）│
│  导航     │  （页面主体）             │  · 📖 功能说明     │
│  240px   │                           │  · 🔗 交互说明     │
│  固定宽度 │                           │  · 📐 业务规则     │
│          │                           │  · ⚠ 待确认项      │
│          │                           │    320px 固定宽度  │
└──────────┴───────────────────────────┴────────────────────┘
```

**左侧导航**：宽 `240px`，固定定位，列出所有功能单元（从`## 功能单元 N`提取），点击切换主内容区，当前选中项左侧有 `3px` 蓝色竖线 + 背景色 `#EBF3FB`。

**主内容区**：`margin-left: 240px`，若标注面板展开则 `margin-right: 320px`，内容最大宽度 `1200px`，`padding: 24px`。

**标注面板**：宽 `320px`，默认**折叠**（隐藏，仅顶部出现"查看标注"按钮），展开后固定在右侧，`z-index: 100`，内含 4 个分区（见第六章）。

**顶部状态栏**：高 `48px`，显示产品名、模块名、文档状态（🤖 AI草稿 / 🔄 PM迭代中 / ✅ 可交付开发），右侧显示"查看标注"按钮。

---

### 2.4 移动端页面整体布局

移动端采用**单栏布局**，取消左侧导航，改为顶部 Header + 内嵌下拉菜单切换功能单元。

```
┌─────────────────────────────┐
│  [产品名] · [模块名]   ≡ 标注│  ← Header 固定 48px
├─────────────────────────────┤
│  当前功能：用户管理  [切换▼] │  ← 功能单元切换栏 40px
├─────────────────────────────┤
│                             │
│  主内容区（单栏滚动）        │  ← 占满宽度
│  padding: 12px              │
│                             │
└─────────────────────────────┘
```

**Header（固定顶部）**：高 `48px`，左侧产品名+模块名（最多显示12字，超出省略），右侧"标注"文字按钮（点击从底部滑出标注抽屉）。

**功能单元切换栏**：高 `40px`，显示当前功能单元名称 + 下拉箭头，点击弹出 `<select>` 或自定义下拉列表，背景 `#F5F7FA`。

**主内容区**：`padding: 12px`，不限最大宽度，所有卡片宽度 `100%`。

**标注抽屉（移动端）**：从底部向上滑出（CSS `transform: translateY`），高度最大 `80vh`，内部可滚动，包含与 PC 端相同的 4 个分区。

**移动端触控规范**：

- 所有可点击元素最小触控区域 `44px × 44px`（符合 Apple HIG / Google Material 规范）
- 按钮高度最小 `40px`，表格行高最小 `48px`
- 禁用 hover 相关样式（移动端不需要）
- 表单输入框 `font-size` 不低于 `16px`（避免 iOS Safari 自动放大）

### 2.5 设计风格规范

**色彩系统**（政务系统风格）：

| 用途 | 色值 |
|---|---|
| 主色（蓝） | `#1A6EBF` |
| 次色（浅蓝背景） | `#EBF3FB` |
| 成功/启用 | `#52C41A` |
| 警告/待确认 | `#FA8C16` |
| 危险/停用 | `#FF4D4F` |
| 边框 | `#E0E0E0` |
| 表格斑马纹 | `#FAFAFA` |
| 文字主色 | `#333333` |
| 文字次要 | `#888888` |
| 背景 | `#F5F7FA` |

**字体**：`font-family: -apple-system, 'Microsoft YaHei', sans-serif`

**圆角**：卡片 `8px`，按钮 `4px`，输入框 `4px`，对话框 `8px`

**间距基准**：`8px`（8的倍数：8/16/24/32）

**动画时效**：弹窗出现 `200ms ease-out`，侧边抽屉滑入 `250ms ease-out`，按钮状态切换 `100ms`

**层叠规则（z-index）**：

| 层级 | z-index | 用途 |
| --- | --- | --- |
| 基础内容 | 1 | 普通页面元素 |
| 标注面板 | 100 | 右侧/底部标注区 |
| 状态切换器 | 110 | 角色/状态演示控件 |
| 模态框遮罩 | 200 | 灰色半透明遮罩 |
| 模态框本体 | 201 | 弹窗内容 |
| Toast 提示 | 300 | 全局成功/错误提示 |

### 2.6 PC 端组件规范

将 ASCII 原型中的元素映射为以下 HTML 组件：

| ASCII 元素 | HTML 组件 | 样式规格 |
| --- | --- | --- |
| `[主操作按钮]` | `<button class="btn-primary">` | 背景 `#1A6EBF`，白字，高 `32px`，padding `0 16px` |
| `[次操作按钮]` | `<button class="btn-secondary">` | 白底，蓝边 `1px solid #1A6EBF`，蓝字，同高 |
| `[危险按钮]` 停用/删除 | `<button class="btn-danger">` | 背景 `#FF4D4F`，白字 |
| `[输入框]` / `___` | `<input type="text">` | 高 `32px`，border `1px solid #E0E0E0`，padding `0 8px` |
| `[日期选择]` | `<input type="date">` | 同输入框规格 |
| `[下拉 ▼]` | `<select>` | 同输入框规格，appearance 保留原生箭头 |
| `□ 选项` | `<input type="checkbox">` | 16px 正方形，选中色 `#1A6EBF` |
| `● 选中 ○ 未选中` | `<input type="radio">` | 16px 圆形，选中色 `#1A6EBF` |
| `┌──┬──┐` 表格 | `<table>` | 表头背景 `#F5F7FA`，奇偶行斑马纹，行高 `48px` |
| `▼ 折叠标题` | `<details><summary>` | summary 含左侧三角图标 |
| `[Tab1] [Tab2]` | div 模拟 Tab 条 | 选中 Tab 底部 `2px solid #1A6EBF`，文字蓝色 |
| `✅启用` | `<span class="badge-success">` | 绿底白字，padding `2px 8px`，圆角 `4px` |
| `⏸停用` / `❌` | `<span class="badge-danger">` | 灰底灰字（`#888`），同规格 |
| `⚠待审核` | `<span class="badge-warning">` | 橙底白字，同规格 |
| `🔍` 搜索 | `<input>` + `<button>` 图标 | 搜索框 `200px`，图标按钮紧贴右侧，无边框 |
| loading 状态 | 按钮 `disabled` + CSS 旋转圆 | `opacity: 0.6`，文字变"提交中..." |
| 左右分栏 `│` | CSS `display: grid` | 列宽按 ASCII 比例设定，gap `16px` |
| 步骤条 | `<ol class="steps">` | 圆圈编号 + 横线连接，当前步骤蓝色 |

### 2.7 移动端组件适配规则

在 `@media (max-width: 767px)` 中覆盖以下样式：

| 组件 | 移动端适配规则 |
| --- | --- |
| 按钮 | 高度改为 `40px`，文字 `14px`，全宽 `width: 100%`（工具栏按钮除外） |
| 输入框 | 高度改为 `40px`，`font-size: 16px`（防止 iOS 放大） |
| 表格 | 改为卡片列表（每行数据变为一张卡片），隐藏不重要列 |
| 弹窗 | 改为从底部滑出的抽屉（bottom sheet），高度 `auto` 最高 `90vh` |
| 左右分栏 | 改为上下堆叠（`grid-template-columns: 1fr`） |
| Tab 条 | 水平滚动（`overflow-x: auto`），不换行 |
| 搜索框 | 宽度 `100%` |
| 状态徽章 | 尺寸保持，但行内排列改为右对齐 |

**表格转卡片示例结构**：

```html
<!-- 移动端：表格行转卡片 -->
<div class="mobile-card">
  <div class="card-row">
    <span class="label">用户名</span>
    <span class="value">张三</span>
  </div>
  <div class="card-row">
    <span class="label">角色</span>
    <span class="value">工作人员</span>
  </div>
  <div class="card-actions">
    <button class="btn-secondary">编辑</button>
    <button class="btn-danger">停用</button>
  </div>
</div>
```

## 三、表单字段规范

### 3.1 字段类型对照

从功能设计文档 `### 功能点描述` 的字段清单中，按字段名推断类型：

| 字段特征 | 字段类型 | HTML 组件 | 校验规则 |
| --- | --- | --- | --- |
| 名称、标题、姓名 | 短文本 | `<input type="text">` | 必填，最大 50 字 |
| 描述、说明、备注 | 长文本 | `<textarea>` rows=3 | 选填，最大 500 字 |
| 日期、时间 | 日期 | `<input type="date">` | 必填，不可晚于今日（结束日期） |
| 日期范围 | 日期区间 | 两个 `<input type="date">` | 开始 ≤ 结束 |
| 状态、类型、分类 | 枚举 | `<select>` | 必填，默认"请选择" |
| 是否、启用/停用 | 布尔 | `<input type="radio">` 是/否 | 必填 |
| 多选类别、权限点 | 多选 | `<input type="checkbox">` 组 | 至少选一项 |
| 手机号 | 电话 | `<input type="tel">` | 11位数字，正则校验 |
| 身份证 | 证件号 | `<input type="text">` | 18位，格式校验 |
| 金额、数量 | 数字 | `<input type="number">` | ≥ 0，保留2位小数（金额） |
| 上传文件/图片 | 文件 | `<input type="file">` | 标注允许类型和大小上限 |

### 3.2 必填标注规则

- 必填字段 `label` 前加红色星号：`<span style="color:#FF4D4F">*</span>`
- 提交时校验：空值 → 输入框 `border-color: #FF4D4F` + 下方红字提示 `"该字段不能为空"`
- 格式错误（如手机号格式）→ 提示具体格式要求

### 3.3 字段状态

每个表单字段需支持以下 4 种视觉状态（通过 CSS class 切换）：

| 状态 | class | 视觉表现 |
| --- | --- | --- |
| 默认 | `field-default` | 边框 `#E0E0E0` |
| 聚焦 | `field-focus` | 边框 `#1A6EBF`，外发光 `0 0 0 2px #EBF3FB` |
| 错误 | `field-error` | 边框 `#FF4D4F`，下方红字提示 |
| 禁用 | `field-disabled` | 背景 `#F5F7FA`，文字 `#888`，cursor not-allowed |

## 四、交互实现规则

### 4.1 基础交互（默认实现）

以下交互**无需在文档中说明**，默认实现：

- **Tab 切换**：点击 Tab 标签切换内容区，选中 Tab 底部蓝线高亮，过渡 `100ms`
- **表格行选中**：点击行背景变 `#EBF3FB`，右侧详情面板联动更新
- **弹窗**：点击"新建"/"编辑"按钮弹出模态框（居中，遮罩 `rgba(0,0,0,0.45)`），点击取消或遮罩关闭
- **表单校验提示**：必填字段为空时，提交后输入框红色边框 + 下方 `12px` 红字提示
- **按钮 loading**：点击提交类按钮后，按钮变 `disabled`，文字改"提交中..."，加旋转圆圈，`2s` 后恢复（模拟）
- **Toast 提示**：操作成功后右上角出现绿色 Toast（"操作成功"），`3s` 后自动消失

### 4.2 文档中明确说明的交互

从`**交互说明**`表格读取，按以下规则实现：

| 交互行为描述 | 实现方式 |
| --- | --- |
| 点击后弹出弹窗 | `onclick` 显示对应 `<div class="modal">` + 遮罩层 |
| 立即生效 | 操作后状态标签即时更新，无 loading（DOM 直接改） |
| 强制下线/立即失效 | 操作确认后，列表该行状态变"停用"（`badge-danger`） |
| 展开行内详情 | 点击行下方插入展开的 `<tr class="detail-row">` |
| 联动变化（字段联动） | 监听 `change` 事件，`display: none/block` 切换相关字段 |
| 防止重复提交 | 点击后按钮 `disabled`，避免二次点击 |
| 确认弹窗（二次确认） | 操作前弹出简单 confirm 弹窗，含说明文字 + 确认/取消按钮 |
| 下载/导出 | 按钮点击后模拟 `<a download>` 触发，或显示"导出中..."提示 |

### 4.3 状态演示控件

若文档包含`**页面状态说明**`，在**功能单元内容区右上角**（标题行右侧）添加状态切换器：

```html
<!-- PC 端：内嵌在内容区标题栏右侧 -->
<div class="state-switcher">
  演示状态：
  <button onclick="setState('default')" class="active">默认</button>
  <button onclick="setState('loading')">加载中</button>
  <button onclick="setState('empty')">空数据</button>
  <button onclick="setState('error')">异常</button>
</div>
```

移动端：状态切换器改为水平滚动的小胶囊按钮条，置于功能单元切换栏下方。

各状态实现要求：

| 状态 | 实现内容 |
| --- | --- |
| 默认 | 正常展示 3-5 条模拟数据 |
| 加载中 | 表格区域显示骨架屏（3行灰色矩形动画） |
| 空数据 | 显示空状态图（圆圈+文字"暂无数据"，居中） |
| 异常 | 显示错误提示框（红色边框，"加载失败，请刷新重试"） |

### 4.4 模拟数据规范

| 数据类型 | 填充规则 |
| --- | --- |
| 表格行数 | 至少 5 行，数据贴合业务场景 |
| 姓名/账号 | 使用文档中已出现的示例（如"张三 zs001"），不足时补充"李四 ls002""王五 ww003" |
| 日期 | 使用固定日期，格式 `YYYY-MM-DD`，时间序列合理（如不能结束早于开始） |
| 状态分布 | 列表中至少包含 2 种不同状态（如已启用/已停用），便于演示状态样式 |
| 数字/金额 | 使用真实量级的数字（如案件数用 "127"，不用 "1"） |
| 图表 | 纯 CSS 矩形色块占位，标注"图表区域（实际集成 ECharts）" |
| 地图 | 灰色矩形 400px 高，内标"区域地图（实际集成 GIS 组件）" |

## 五、功能流程图规则

### 5.1 渲染方案

使用 **Mermaid.js**（CDN 引入），这是本规则唯一允许的外部依赖。原因：开发环境有网络，Mermaid 内联版本约 2MB 过于臃肿。

```html
<!-- 在 <head> 中引入，仅此一处外部依赖 -->
<script src="https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js"></script>
<script>mermaid.initialize({ startOnLoad: true, theme: 'default' });</script>
```

### 5.2 流程图放置位置

每个功能单元主内容区，在**功能单元标题和页面操作区之间**插入流程图，折叠展示：

```
┌────────────────────────────────────────────┐
│  用户管理                    [新建用户 +]   │  ← 功能单元标题行
├────────────────────────────────────────────┤
│  ▶ 功能流程图（点击展开）                  │  ← 默认折叠
├────────────────────────────────────────────┤
│  [全部用户] [内部用户] ...                  │  ← 主操作区
└────────────────────────────────────────────┘
```

使用 `<details><summary>` 实现折叠，默认 `closed`（不影响主操作区的第一视线）。

### 5.3 Mermaid 图类型选择

根据功能单元特征，选择对应图类型：

| 功能特征 | 图类型 | Mermaid 关键字 |
| --- | --- | --- |
| 操作步骤明确（新建→填写→提交→审核） | 流程图 | `flowchart TD` |
| 涉及多角色协作（发起人/审核人/系统） | 泳道图 | `sequenceDiagram` 或 `flowchart TD` + subgraph |
| 状态流转（启用/停用/待审核） | 状态机图 | `stateDiagram-v2` |
| 数据查询类（无明显流程） | 跳过，不生成流程图 | — |

### 5.4 流程图内容生成规则

AI 从以下来源推导流程图内容，**不凭空创造**：

1. `**交互说明**` 表格中的操作链路
2. `**业务规则**` 中的前置条件和约束
3. `**页面状态说明**` 中的状态流转描述
4. ASCII 原型中按钮的操作顺序

**节点命名规则**：使用文档中的原始术语，不重新命名；操作节点用矩形，判断节点用菱形，终态用圆角矩形。

## 六、多功能单元处理规则

当一个功能设计文档包含多个`## 功能单元`时：

1. **每个功能单元**对应左侧导航中的一个菜单项（PC）/ 下拉选项（移动端）
2. **默认展示**第一个功能单元
3. **切换方式**：点击左侧导航项，主内容区切换，URL hash 变化（`#unit-1`）
4. **功能单元间关联**：若文档中提到跳转关系（如"点击新建跳转到表单页"），在同一功能单元内用模态框或内嵌面板实现，不跨单元跳转
5. **标注面板联动**：切换功能单元时，标注面板内容同步更新为当前单元的功能说明和交互规则

## 七、特殊模块处理规则

### 7.1 包含图表的模块（如统计分析报告）

- 使用**纯 CSS 柱状图/折线图**占位，不引入 ECharts 等外部库
- 图表区域标注灰色水印：`"图表区域（实际集成 ECharts）"`
- 提供图表类型说明标签（如"趋势图""区域热力图"）
- 移动端：图表区域高度改为 `200px`，宽度 `100%`

### 7.2 包含地图的模块（如区域风险预警）

- 用一个灰色矩形区域代替地图，内部标注：`"沈阳市区域地图（实际集成 GIS 组件）"`
- 在地图旁边提供图例（四色预警色块 + 文字说明）
- 移动端：地图置于图例上方，图例改为横向排列

### 7.3 包含权限差异的模块（如数据查询门户）

- PC 端：在顶部状态栏右侧添加**角色切换器**：`当前角色：[内部工作人员 ▼]`
- 移动端：角色切换器置于 Header 标题下方一行
- 切换角色后，页面中受权限控制的按钮/菜单项相应显示/隐藏

### 7.4 包含流程审批的模块

- 用**步骤条（Step Bar）**展示流程阶段
- 当前步骤高亮蓝色，已完成步骤绿色打勾，未来步骤灰色
- 移动端：步骤条改为垂直排列（`flex-direction: column`）

## 八、标注面板规则

### 8.1 标注面板结构

标注面板是原型的"开发说明书区域"，包含 4 个分区，**按以下顺序排列**：

```
📋 开发参考面板
├── 📖 功能说明（第一分区，最重要）
├── 🔗 交互说明
├── 📐 业务规则
└── ⚠ 待确认项
```

### 8.2 📖 功能说明（第一分区，可编辑可保存）

**来源**：提取功能设计文档中每个`## 功能单元 N`下的**功能概述/目标**部分。

**放置规则**：

- 标注面板展开后，第一个可见分区即为"功能说明"
- 内容为该功能单元的目标描述、使用场景、涉及角色（1-3句话）
- 切换功能单元时，功能说明内容联动更新

**可编辑 + 本地保存规则**：

- 描述内容区域添加 `contenteditable="true"`，点击后可直接编辑
- 右上角显示"✏ 编辑"按钮，点击后边框高亮提示进入编辑态，再次点击变"💾 保存"
- 点击"保存"后：
  1. 内容写入 `localStorage`，key 格式为 `proto_desc_[模块名]_[功能单元序号]`
  2. 右上角 Toast 提示"已保存"（绿色，2s 消失）
  3. 按钮恢复"✏ 编辑"
- 页面加载时：自动从 `localStorage` 读取并填充，若无则显示从文档提取的原始内容

**HTML 示例**：

```html
<div class="annotation-section">
  <div class="section-header">
    <span class="section-title">📖 功能说明</span>
    <button class="edit-btn" onclick="toggleEditDesc('unit-1')">✏ 编辑</button>
  </div>
  <div class="feature-desc"
       id="desc-unit-1"
       contenteditable="false"
       data-storage-key="proto_desc_用户管理_1">
    <p><strong>功能目标：</strong>支持系统管理员对平台用户进行统一管理，包括创建账号、分配角色、启用/停用账号。</p>
    <p><strong>使用场景：</strong>新员工入职配置账号；人员离职后快速停用账号。</p>
    <p><strong>涉及角色：</strong>系统管理员</p>
  </div>
</div>

<script>
function toggleEditDesc(unitId) {
  const el = document.getElementById('desc-' + unitId);
  const btn = el.previousElementSibling.querySelector('.edit-btn');
  const isEditing = el.contentEditable === 'true';
  if (isEditing) {
    el.contentEditable = 'false';
    el.style.border = '';
    localStorage.setItem(el.dataset.storageKey, el.innerHTML);
    btn.textContent = '✏ 编辑';
    showToast('已保存');
  } else {
    el.contentEditable = 'true';
    el.style.border = '1px solid #1A6EBF';
    el.style.borderRadius = '4px';
    el.focus();
    btn.textContent = '💾 保存';
  }
}
// 页面加载时读取本地存储
document.querySelectorAll('.feature-desc[data-storage-key]').forEach(el => {
  const saved = localStorage.getItem(el.dataset.storageKey);
  if (saved) el.innerHTML = saved;
});
</script>
```

### 8.3 🔗 交互说明

来自功能设计文档`**交互说明**`表格，逐条列出，保留原始描述，不删改。

### 8.4 📐 业务规则

来自功能设计文档`**业务规则**`章节，提取关键约束条目（如字段唯一性、状态流转限制、权限约束等）。

### 8.5 ⚠ 待确认项

来自`**待补充内容**`和 AC 编号列表，以橙色标注：

```html
<div class="pending-item">
  ⚠ [AC-08] 是否对接政务统一认证（SSO）？若需对接，账号创建流程调整
</div>
```

**无待确认项时**：显示 `<p style="color:#888">暂无待确认项</p>`，不显示空白区域。

## 十一、生成指令模板

> **⚠ 执行约束：每次只生成一个功能模块的原型。**
> 不得在同一次响应中批量输出多个模块的 HTML。原因：单模块已足够复杂，批量生成会导致质量下降、细节遗漏、上下文超限。
