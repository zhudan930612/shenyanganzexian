## 一、本版修复（全盘审查结论）

### 1. 关键问题

| 编号 | 问题 | 影响 |
|---|---|---|
| P1 | `outline` 为整段字符串，下游却引用 `{outline-①}` 这类“隐式子字段” | 分支节点拿不到稳定输入，生成结果漂移 |
| P2 | 节点2a输出是多行文本，节点2c却按 `|` 分隔解析 | 检索词拆解失效，导致知识库召回错误 |
| P3 | 多处变量名未定义（如 `{seg_2核心事实}`、`{kb5_result-第1条}`、`{outline-④⑤}`） | 运行时报空值或错误引用 |
| P4 | 部分提示词写法为“同上格式”，缺少独立输出契约 | 节点行为不稳定，难以排错 |
| P5 | 提示词结构不统一 | 维护成本高，协作理解成本高 |

### 2. 修复策略

1. 增加“结构化大纲拆分节点”（节点4b），将 `outline_raw` 拆成 `outline_1`~`outline_5` 与 `outline_core_2`~`outline_core_5`。  
2. 统一检索词输出标签与解析逻辑（节点2a ↔ 节点2c 一一对应）。  
3. 所有分支节点只接收“显式变量”，不再使用隐式占位符。  
4. 全部LLM提示词统一为 5 段：`角色、任务、执行流程、限制、输出示例`。  
5. 每个节点都给出明确输入、输出与失败兜底文案。

### 节点2c：检索词拆解（代码节点）

**输入**：`analysis_keywords`  
**输出**：`kw_kb1` / `kw_kb2` / `kw_kb3` / `kw_kb4` / `kw_kb5`

```python
import re


def _pick(text: str, label: str) -> str:
    m = re.search(rf"{label}[:：]\s*(.*)", text or "")
    return m.group(1).strip() if m else ""


def _parse(raw: str) -> dict:
    return {
        "kw_kb1": _pick(raw, "合规红线检索词"),
        "kw_kb2": _pick(raw, "核心概念检索词"),
        "kw_kb3": _pick(raw, "法规条文检索词"),
        "kw_kb4": _pick(raw, "产品规则检索词"),
        "kw_kb5": _pick(raw, "司法案例检索词"),
    }


# 兼容两种代码节点入参风格：
# 1) main(analysis_keywords="...")
# 2) main(content={...})
def main(analysis_keywords: str = "", analysis_k: str = "", content: dict = None, **kwargs):
    content = content or {}
    raw = (
        analysis_keywords
        or analysis_k
        or content.get("analysis_keywords", "")
        or content.get("analysis_k", "")
        or kwargs.get("analysis_keywords", "")
        or kwargs.get("analysis_k", "")
        or ""
    )
    return _parse(raw)
```

### 节点4b：大纲结构化拆分（代码节点）

**输入**：`outline_raw`  
**输出**：`outline_1`~`outline_5` / `outline_core_2`~`outline_core_5`

```python
import re


def _pick(text: str, label: str, default: str = "") -> str:
    m = re.search(rf"{label}[:：]\s*(.*)", text)
    return m.group(1).strip() if m else default


def main(content: dict) -> dict:
    raw = content.get("outline_raw", "") or ""

    return {
        "outline_1": _pick(raw, "大纲①"),
        "outline_2": _pick(raw, "大纲②"),
        "outline_3": _pick(raw, "大纲③"),
        "outline_4": _pick(raw, "大纲④"),
        "outline_5": _pick(raw, "大纲⑤", "无"),
        "outline_core_2": _pick(raw, "核心结论②"),
        "outline_core_3": _pick(raw, "核心结论③"),
        "outline_core_4": _pick(raw, "核心结论④"),
        "outline_core_5": _pick(raw, "核心结论⑤", "无"),
    }
```
