# 生产安全事故统计分析与智能预警系统 · 功能清单（工时评估用）

| 字段 | 内容 |
|------|------|
| 文档用途 | 供开发团队评估各功能工时；功能描述和备注由 PM 填写，工时栏由开发填写 |
| 生成日期 | 2026-03-24 |
| 版本状态 | 草稿，待开发评估后确认 |

> **说明**
> - 工时单位为**人天**
> - 预警模块存在路径决策（路径一：浙大模型 / 路径二：基础算法），路径确认前建议两条路径分别报价
> - 🔴 标注为评估前需优先确认的前置问题，未确认前对应功能工时存在较大不确定性

| 模块 | # | 功能点 | 功能描述 | 前端工时 | 后端工时 | 备注/依赖 |
|------|---|--------|----------|----------|----------|-----------|
| **首页 / 驾驶舱** | 0.1 | 局领导首页 | 登录后展示：全市事故核心指标（本月/本年起数、同比变化、死亡人数）；当前预警汇总（四色各几条，点击跳转详情）；区域风险地图缩略图（四色标注）；近12个月事故趋势折线图 | | | 地图组件复用 3.4；指标口径需与甲方业务专家确认 |
| **首页 / 驾驶舱** | 0.2 | 工作人员首页（事故调查科） | 登录后展示：待处置预警数量及快捷入口；今日数据同步状态（成功/失败/数据量）；近期报告生成状态；快捷操作区（录入事故 / 生成报告 / 查看预警） | | | 同步状态数据复用 1.6；快捷操作跳转对应功能页 |
| **首页 / 驾驶舱** | 0.3 | 县局工作人员首页 | 登录后展示：本县核心数据指标（本月/本年事故起数、同比）；本县当前预警状态（如有）；快捷操作区（录入事故 / 导入数据）；仅展示本县数据，后端行级隔离 | | | 数据隔离逻辑复用 1.2/5.2 |
| **数据接入与治理** | 1.1 | 数据自动同步（直报系统对接） | 每日凌晨定时拉取直报系统前一日新增/变更数据，增量写入数据仓库；对接失败触发告警 | | | 🔴 数据获取方式未确认（API/自动化脚本/手动），三种方案工时差异较大 |
| **数据接入与治理** | 1.2 | 事故上报表单（手动录入） | 县应急局工作人员通过 Web 表单手动录入事故数据；字段/校验/枚举与直报系统一致；各县只能录入本县数据 | | | 🔴 字段清单依赖甲方数据字典；区县权限隔离为必须的后端能力 |
| **数据接入与治理** | 1.3 | 事故数据模板导入（Excel/CSV 批量） | 县应急局工作人员通过标准模板批量导入事故数据；执行字段/枚举/重复校验；失败输出错误明细；成功数据入库打批次标识；导入前需填写 Summary（required）说明本次更新内容 | | | 模板口径与直报系统一致；单次导入量级上限待确认；Summary 为空时禁止导入 |
| **数据接入与治理** | 1.4 | 本县事故明细查询（县局用户） | 县局工作人员查询本县全量事故明细（含历史），支持时间/事故类型/等级筛选；仅展示明细列表，不含统计图表和报告 | | | 数据行级隔离在后端实现，各县只能查本县 |
| **数据接入与治理** | 1.5 | 数据清洗与治理（后台自动） | 对同步/录入数据自动标准化：字段格式统一、去重、异常值标注、缺失字段补全；结果写入数据仓库 | （监控面板） | | 🔴 清洗规则依赖甲方数据字典，字典未到位前工时难以精确评估 |
| **数据接入与治理** | 1.6 | 同步状态监控与告警 | 管理员查看每日同步任务状态（成功/失败/数据量/耗时）；同步失败或数据量异常时自动告警 | | | 告警通知方式（系统内/邮件/短信）待甲方确认 |
| **统计分析报告** | 2.1 | 报告生成 · 月度简报 | 工作人员选择月度模板和时间范围，系统自动聚合数据渲染核心指标图表（1页纸摘要）；支持预览 | | | 图表样式需甲方确认；依赖 2.4 同比环比计算 |
| **统计分析报告** | 2.2 | 报告生成 · 季度深度分析 | 工作人员选择季度模板，系统生成含趋势图、行业对比、同比环比标注的图文报告；支持预览 | | | 依赖 2.4 同比环比计算 |
| **统计分析报告** | 2.3 | 报告生成 · 年度白皮书 | 工作人员选择年度模板，系统生成全维度数据汇总报告，含多行业、多区域年度趋势和横向对比；支持预览 | | | 需确认历史数据年限范围 |
| **统计分析报告** | 2.4 | 同比 / 环比自动计算与标注 | 所有图表自动计算并标注同比（去年同期）和环比（上一周期）变化率；颜色编码：红色上升/绿色下降；数据缺失时降级展示 | | | 通用基础计算能力，被 2.1/2.2/2.3 共用 |
| **统计分析报告** | 2.5 | 多维度下钻 | 报告中点击任意图表指标展开该维度详细分布（如点击"建筑业"下钻事故类型/时段/区域），最多 3 层 | | | 前端交互逻辑较复杂，工时较重；下钻维度需与甲方确认 |
| **统计分析报告** | 2.6 | 文字解读生成 | 每张图表自动生成一段解读文字；路径一（浙大）：AI 动态生成，需人工审核后发布；路径二（基础算法）：规则模板填充，无动态分析结论 | （含审核流程界面） | （路径一/二分别报价） | 🔴 工时依赖路径决策；路径一需与浙大接口联调 |
| **统计分析报告** | 2.7 | 异常点自动标注 | 趋势图中自动识别并标注偏离正常区间的时间节点；鼠标悬停弹出该时段具体事故信息 | | | 异常阈值规则需与甲方业务专家共同定义 |
| **统计分析报告** | 2.8 | 报告导出（Word / PDF） | 工作人员确认后一键导出 Word 或 PDF，排版符合政府公文规范 | | | 政府公文排版要求需甲方提供样例；渲染引擎选型影响工时 |
| **事故风险预警** | 3.0 | 预警模型配置 | 管理员通过配置界面选择当前启用的预警算法：①趋势外推法（时间序列预测）；②相似度匹配法（寻找最相似往年同期参考后续走势）；③因素关联法（结合气象/节假日/企业生产负荷等外部数据，进阶选项）；支持对各算法参数进行配置（如时间窗口、权重系数）；配置变更实时生效并记录操作日志 | | | 🔴 因素关联法依赖外部数据源接入（气象/节假日/负荷），接入方式和数据可用性需甲方确认后才能评估工时；建议路径一/路径二分别报价时同步考虑此模块 |
| **事故风险预警** | 3.1 | 预警模型引擎 | 定时触发预警计算；按选定路径执行算法（趋势外推/相似度匹配/因素关联）；输出风险指数；与阈值比对判定预警等级 | （阈值配置界面） | （路径一/二分别报价） | 🔴 路径确认前无法精确评估；路径一需与浙大接口联调；路径二建议历史数据 ≥ 3 年 |
| **事故风险预警** | 3.2 | 总量维度预警 | 预测全市未来一周/一月事故起数是否超历史阈值，触发时生成对应等级预警（蓝/黄/橙/红） | | | 四色阈值标准需甲方安全专家定义 |
| **事故风险预警** | 3.3 | 行业类型维度预警 | 按行业（化工/建筑施工/道路运输等）分别计算风险指数，对风险升高的行业触发分级预警 | | | 行业分类枚举须与直报系统口径一致 |
| **事故风险预警** | 3.4 | 区域维度预警（GIS 地图） | 在地图上对区县级行政区域进行四色预警标注，风险升高区域高亮；支持点击查看区域预警详情 | （GIS 地图集成，工时较重） | | 地图组件建议天地图（政务推荐）；区县行政边界数据需提前准备 |
| **事故风险预警** | 3.5 | 预警原因解释 | 每条预警自动附带结构化原因说明（历史规律 + 近期数据变化），以可读文字呈现，不输出无解释结论；路径一 AI 生成，路径二规则模板 | | | 解释内容须基于可量化数据，不含主观判断 |
| **事故风险预警** | 3.6 | 预警通知推送 | 预警触发后自动推送通知给相关工作人员和局领导（按等级分级，领导仅接收红橙） | （系统内消息中心） | | 通知方式（系统内/邮件/短信/钉钉）待甲方确认，不同方式工时差异较大 |
| **事故风险预警** | 3.7 | 预警处置记录 | 工作人员对预警执行确认/忽略/登记处置措施；系统记录处置状态和意见；事后关联实际数据统计预警准确率 | | | 预警状态流转需明确：待处置 → 处置中 → 已归档 |
| **数据查询（外部部门）** | 4.1 | 外部部门自助查询 | 外部部门用户（交通局/消防/公安等）按权限范围内的条件查询事故统计数据；结果以图表 + 数据表格展示；不展示内部报告/预警/AI解读入口；不含个案敏感信息 | | | 各部门权限范围需甲方提供清单，未确认前界面边界和后端隔离逻辑无法定义 |
| **数据查询（外部部门）** | 4.2 | 查询结果导出 | 外部用户可将查询结果导出为 Excel 或 PDF，仅含权限范围内的聚合数据，不含个案敏感信息 | | | 是否允许导出需甲方明确数据安全策略 |
| **系统管理** | 5.1 | 账号管理 · 内部用户 | 管理员为事故调查科工作人员、局领导创建/编辑/停用账号；分配角色和系统权限 | | | 是否对接政务 SSO 待确认，对接后工时增加 |
| **系统管理** | 5.2 | 账号管理 · 县应急局用户 | 管理员为各区县局工作人员开设账号并绑定所属区县；县级账号自动限制只能操作和查询本县数据 | | | 区县级数据行级隔离为核心后端能力，需在数据访问层统一实现 |
| **系统管理** | 5.3 | 账号管理 · 外部部门用户 | 管理员为外部部门用户创建账号，配置可查询的数据维度（事故类型白名单/区域范围）；权限变更自动记录日志 | | | 各部门权限维度需甲方提供清单后才能精确设计 |
| **系统管理** | 5.4 | 登录认证 | 账号密码登录；密码策略配置（复杂度/过期）；可选对接政务 SSO | | | SSO 对接与否待确认；对接需额外工时且依赖甲方信息化部门配合 |
| **系统管理** | 5.5 | 操作日志留痕 | 所有用户的登录/查询/导出/预警处置等操作均自动记录日志（操作人/时间/操作内容/IP）；支持筛选查询和导出 | | | 日志保留年限需满足等保 2.0 要求（通常 ≥ 6 个月）；存储量级需提前规划 |
